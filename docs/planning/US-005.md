## User Story

**Epic:** EP-002 — Ricerca Locale e Scoperta
**Priorità:** HIGH | **Story Points:** 3

**Story**
As genitore in cerca di giocattoli,
I want cercare annunci per parola chiave e filtrarli per fascia d'età,
so that trovo rapidamente opzioni adatte a mio figlio.

**Criteri di Accettazione**
- [ ] La ricerca supporta testo libero su titolo e descrizione degli annunci.
- [ ] Posso applicare e rimuovere il filtro per fascia d'età senza perdere la query testuale.
- [ ] Se non ci sono risultati, il sistema mostra uno stato vuoto chiaro con invito a modificare i filtri.

> **Nota:** Il filtro per "categoria" è escluso dallo scope di questa storia (opzione C concordata con il product owner). Andrà pianificato in una US separata quando il campo `category` sarà definito nel modello.

## Soluzione Tecnica

La soluzione è interamente read-only sul modello dati esistente — nessuna migration necessaria. I query params della URL (`?q=...&ageRange=...`) sono la fonte di verità per lo stato dei filtri, garantendo persistenza su refresh e condivisibilità del link.

- Introdurre `src/lib/toy-ads/search.ts` con la funzione `searchToyAds({ query, ageRange })` che esegue una query Prisma con `where: { status: "PUBLISHED", ...(ageRange && { ageRange }), ...(query && { OR: [{ title: { contains: query, mode: "insensitive" } }, { description: { contains: query, mode: "insensitive" } }] }) }` e ritorna un array di annunci con i campi necessari per la card risultati.
- Creare `src/app/search/page.tsx` come server component pubblico (non protetto da middleware): riceve `searchParams: Promise<{ q?: string; ageRange?: string }>`, awaita i params, chiama `searchToyAds`, costruisce gli URL pubblici delle immagini via `supabase.storage.from(MEDIA_BUCKET).getPublicUrl()` (stesso pattern di `[id]/page.tsx`), e renderizza risultati o stato vuoto.
- Creare `src/app/search/search-form.tsx` come client component (`"use client"`): gestisce l'input testuale e il select fascia d'età, usa `useRouter().push` per aggiornare i query params senza ricaricare la pagina, mostra i chip dei filtri attivi con la possibilità di rimuoverli singolarmente.
- Aggiornare homepage (`src/app/page.tsx`) o layout per aggiungere un link di accesso alla pagina `/search`.

## Strategia di Test

La strategia separa il test della logica di query (unit, senza DB reale) dal comportamento della pagina (integration con mock Prisma), garantendo copertura senza dipendere dall'infrastruttura.

- Unit: testare `searchToyAds` con Prisma mockato — query testuale su titolo, query su descrizione, filtro `ageRange` attivo, combinazione query+filtro, nessun parametro (tutti gli annunci PUBLISHED), caratteri speciali nell'input (apostrofi, virgolette), query che non produce risultati.
- Integration: testare la page `search/page.tsx` con `searchParams` simulati — verifica che i risultati siano renderizzati, verifica stato vuoto quando `searchToyAds` ritorna array vuoto.
- Regression: verificare che la funzione `searchToyAds` filtri sempre e solo annunci con `status: PUBLISHED` indipendentemente dagli altri parametri.
